## Entities

### Video (New — `api/Data/Video.cs`)

Central entity tracking a video through its lifecycle: creation, upload, transcoding, and completion. Maps to the `Videos` table in PostgreSQL.

| Field | CLR Type | Column | Nullable | Description |
|-------|----------|--------|----------|-------------|
| `Id` | `Guid` | `id` | No | Primary key, generated by the application at creation time |
| `UserId` | `Guid` | `user_id` | No | Owner of the video. Uses the placeholder constant until authentication is implemented |
| `BlobPath` | `string` | `blob_path` | No | Azure Blob Storage path (e.g., `uploads/anonymous/{guid}.mp4`) |
| `ContentType` | `string` | `content_type` | No | MIME type of the uploaded video (e.g., `video/mp4`, `video/quicktime`) |
| `FileSizeBytes` | `long` | `file_size_bytes` | No | Size of the original uploaded file in bytes |
| `Status` | `VideoStatus` | `status` | No | Current lifecycle status. Stored as a string in PostgreSQL |
| `CloudflareVideoUid` | `string?` | `cloudflare_video_uid` | Yes | Cloudflare Stream video identifier, set when submitted for transcoding |
| `HlsUrl` | `string?` | `hls_url` | Yes | HLS streaming manifest URL from Cloudflare |
| `DashUrl` | `string?` | `dash_url` | Yes | DASH streaming manifest URL from Cloudflare |
| `ThumbnailUrl` | `string?` | `thumbnail_url` | Yes | Video thumbnail image URL from Cloudflare |
| `Duration` | `double?` | `duration` | Yes | Video duration in seconds, from Cloudflare webhook |
| `InputWidth` | `int?` | `input_width` | Yes | Source video width in pixels, from Cloudflare webhook |
| `InputHeight` | `int?` | `input_height` | Yes | Source video height in pixels, from Cloudflare webhook |
| `ErrorReasonCode` | `string?` | `error_reason_code` | Yes | Error code from Cloudflare if transcoding failed |
| `ErrorReasonText` | `string?` | `error_reason_text` | Yes | Human-readable error description from Cloudflare |
| `CreatedAt` | `DateTimeOffset` | `created_at` | No | Timestamp when record was created, set by application |
| `UpdatedAt` | `DateTimeOffset` | `updated_at` | No | Timestamp of last update, set by application on every save |

**Indexes**:
- Primary key on `id`
- Unique index on `blob_path` (each blob maps to exactly one video)
- Index on `cloudflare_video_uid` (webhook lookup by Cloudflare UID)
- Index on `user_id` (future queries to list videos for a user)
- Index on `status` (future queries to find videos in a given state)

**Validation Rules**:
- `BlobPath` must not be empty. Maximum length: 1024 characters.
- `ContentType` must not be empty. Maximum length: 256 characters.
- `FileSizeBytes` must be greater than 0.
- `Status` is stored as a `text` column (EF Core string conversion of the enum), not as an integer, for readability in direct database queries.
- `CloudflareVideoUid` maximum length: 256 characters.
- URL fields (`HlsUrl`, `DashUrl`, `ThumbnailUrl`) maximum length: 2048 characters.
- `ErrorReasonCode` maximum length: 256 characters.
- `ErrorReasonText` maximum length: 4096 characters.

### VideoStatus (New — `api/Data/VideoStatus.cs`)

Enum representing the video lifecycle states. Stored as a lowercase string in PostgreSQL.

| Value | String | Description |
|-------|--------|-------------|
| `Created` | `created` | Video record created, upload URL issued to client |
| `Uploaded` | `uploaded` | Reserved for future blob-upload-confirmed event. Not set in the initial implementation |
| `Processing` | `processing` | Submitted to Cloudflare Stream for transcoding |
| `Ready` | `ready` | Transcoding complete, playback URLs available |
| `Failed` | `failed` | Transcoding failed, error details stored |

**State Transitions** (initial implementation):

```
Created ──► Processing ──► Ready
                       └──► Failed
```

Valid transitions:
- `Created` → `Processing` (ProcessVideo receives BlobCreated event, submits to Cloudflare, stores UID)
- `Processing` → `Ready` (CloudflareWebhook receives readyToStream=true)
- `Processing` → `Failed` (CloudflareWebhook receives error status)

> **Future**: The `Uploaded` status is defined in the enum and reserved for a
> future blob-upload-confirmed event. When implemented, the transitions will
> become: Created → Uploaded → Processing → Ready/Failed.

Note: Idempotent re-processing is allowed — a webhook for an already-`Ready` or already-`Failed` video will update the record to the new status from Cloudflare.

## DbContext Configuration

### StandupDbContext (New — `api/Data/StandupDbContext.cs`)

EF Core `DbContext` for the Standup database. Registered via `IDbContextFactory<StandupDbContext>` in dependency injection.

**DbSets**:

| Property | Type | Description |
|----------|------|-------------|
| `Videos` | `DbSet<Video>` | Access to the Videos table |

**OnModelCreating Configuration** (fluent API):

```
Video entity:
  - ToTable("Videos")
  - HasKey(v => v.Id)
  - Property(v => v.Id).ValueGeneratedNever()
  - Property(v => v.BlobPath).IsRequired().HasMaxLength(1024)
  - Property(v => v.ContentType).IsRequired().HasMaxLength(256)
  - Property(v => v.Status).HasConversion<string>().IsRequired()
  - Property(v => v.CloudflareVideoUid).HasMaxLength(256)
  - Property(v => v.HlsUrl).HasMaxLength(2048)
  - Property(v => v.DashUrl).HasMaxLength(2048)
  - Property(v => v.ThumbnailUrl).HasMaxLength(2048)
  - Property(v => v.ErrorReasonCode).HasMaxLength(256)
  - Property(v => v.ErrorReasonText).HasMaxLength(4096)
  - HasIndex(v => v.BlobPath).IsUnique()
  - HasIndex(v => v.CloudflareVideoUid)
  - HasIndex(v => v.UserId)
  - HasIndex(v => v.Status)
```

**Column Naming Convention**: EF Core's Npgsql provider applies `snake_case` naming by default when using `UseSnakeCaseNamingConvention()` from the `EFCore.NamingConventions` package. However, to avoid an extra dependency, column names will be configured explicitly via the `[Column("snake_case")]` attribute or fluent API `.HasColumnName()`.

## DI Registration Pattern

### Program.cs Changes

The following services are added to `HostBuilder` configuration:

1. **Build `NpgsqlDataSource`** with managed identity token provider:
   ```
   NpgsqlDataSourceBuilder
     → ConnectionString from POSTGRESQL_HOST, POSTGRESQL_DATABASE, POSTGRESQL_USERNAME env vars
     → UsePeriodicPasswordProvider (DefaultAzureCredential, scope, refresh interval)
     → Build()
   ```

2. **Register `IDbContextFactory<StandupDbContext>`**:
   ```
   services.AddDbContextFactory<StandupDbContext>(options =>
       options.UseNpgsql(dataSource))
   ```

3. **Register singleton `NpgsqlDataSource`** so it is disposed with the host.

## Relationships

```
Client (iPhone App)
    │
    │ POST /api/video
    ▼
CreateVideo Function
    │ Creates Video(status=created, userId=placeholder)
    │ Returns VideoId + UploadUrl
    ▼
Azure Blob Storage ◄── Client uploads video file
    │
    │ BlobCreated EventGrid event
    ▼
ProcessVideo Function
    │ Looks up Video by BlobPath
    │ Updates status: created → uploaded
    │ Submits to Cloudflare Stream (meta includes videoId + blobPath)
    │ Updates status: uploaded → processing
    │ Stores CloudflareVideoUid
    ▼
Cloudflare Stream ── transcodes video ── fires webhook
    │
    │ POST /api/webhooks/cloudflare/stream (meta round-trips videoId)
    ▼
CloudflareWebhook Function
    │ Looks up Video by meta.videoId (primary) or CloudflareVideoUid (fallback)
    │ Updates status: processing → ready (or failed)
    │ Stores HLS/DASH/thumbnail URLs, duration, dimensions
    ▼
PostgreSQL ◄── Video record with full lifecycle data
```

## PostgreSQL Table DDL (Reference)

The following is the expected SQL generated by the EF Core migration. It is provided for reference; the actual migration is authoritative.

```sql
CREATE TABLE "Videos" (
    id uuid NOT NULL,
    user_id uuid NOT NULL,
    blob_path character varying(1024) NOT NULL,
    content_type character varying(256) NOT NULL,
    file_size_bytes bigint NOT NULL,
    status text NOT NULL,
    cloudflare_video_uid character varying(256),
    hls_url character varying(2048),
    dash_url character varying(2048),
    thumbnail_url character varying(2048),
    duration double precision,
    input_width integer,
    input_height integer,
    error_reason_code character varying(256),
    error_reason_text character varying(4096),
    created_at timestamp with time zone NOT NULL,
    updated_at timestamp with time zone NOT NULL,
    CONSTRAINT "PK_Videos" PRIMARY KEY (id)
);

CREATE UNIQUE INDEX "IX_Videos_blob_path" ON "Videos" (blob_path);
CREATE INDEX "IX_Videos_cloudflare_video_uid" ON "Videos" (cloudflare_video_uid);
CREATE INDEX "IX_Videos_user_id" ON "Videos" (user_id);
CREATE INDEX "IX_Videos_status" ON "Videos" (status);
```
